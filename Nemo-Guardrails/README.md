# ARIA — NeMo Guardrails Deep-Dive

Securing a vulnerable LLM banking agent with NVIDIA NeMo Guardrails.

**Week 6 — The Gen Academy**

ARIA is a mock support agent for "NeoBank" with real tools (policy lookup +
account query) and a knowledge base that contains sensitive internal data. We
run an attack suite against the unguarded agent to prove it leaks, then
progressively apply guardrail configs and re-run the same attacks to show the
before/after difference.

## Project layout

```
aria/                     # The reusable Python package (all the plumbing)
  knowledge_base.py       # Mock KB, customers, and transactions
  tools.py                # lookup_policy / query_account + OpenAI tool schema
  prompts.py              # ARIA system prompt + guardrail instructions
  agent.py                # Unguarded baseline agent (OpenAI function calling)
  attacks.py              # ATTACK_SUITE + rail-specific attack lists
  guardrails.py           # create_config() and test_rail() helpers
  config.py               # .env loading + API-key checks
guardrails_configs/       # Config directories generated by the notebook
nemo_guardrails_deepdive.ipynb   # The demo — just the guardrails story
.env.example              # Template for your API keys
```

The notebook holds **only the NeMo Guardrails demo**. Everything reusable
(data, tools, prompts, the agent, attack suites, helpers) lives in `aria/` and
is imported.

## Setup

This is a [uv](https://docs.astral.sh/uv/) project (requires Python 3.10+).

```bash
# Install dependencies (including notebook tooling)
uv sync --extra notebook

# Optional: local jailbreak-detection heuristics (Config 3, ~3GB torch)
uv sync --extra notebook --extra heuristics

# Add your API keys
cp .env.example .env
# then edit .env and fill in OPENAI_API_KEY and NVIDIA_API_KEY
```

## Running the demo

```bash
uv run jupyter notebook nemo_guardrails_deepdive.ipynb
```

Or select the `.venv` interpreter in VS Code / Cursor and run the notebook cells.

## API keys

| Key | Used for | Where to get it |
|---|---|---|
| `OPENAI_API_KEY` | Main ARIA LLM + self-check rails | https://platform.openai.com/api-keys |
| `NVIDIA_API_KEY` | NIM rails (Content Safety, GLiNER PII) | https://build.nvidia.com |

Keys are loaded from `.env` via `aria.config.load_environment()`. The `.env`
file is git-ignored; never commit real secrets.

## The configs

| Config | Rail | What it does |
|---|---|---|
| 1 | Self-check input | LLM-as-judge screens the user message before the main LLM |
| 2 | Self-check output | LLM-as-judge screens the bot response before delivery |
| 3 | Jailbreak heuristics | Local perplexity check (no API call) — optional extra |
| 4 | NIM Content Safety | NVIDIA NemoGuard 8B classifies across 23 safety categories |
| 5a | GLiNER PII (detect) | Detect and **block** messages containing PII entities |
| 5b | GLiNER PII (mask) | Detect and **mask** (redact) PII entities in place |

Each config is written to its own directory under `guardrails_configs/`
(`01_self_check_input`, …, `05b_pii_gliner_mask`) by the notebook.
